Die zentrale Entscheidung lautet:

```text
Das MasterTemplate darf keine normale Blogvorlage sein.
Es muss ein Entscheidungs-, Prüf- und Vertrauens-Template sein.
```

Jeder Artikel soll später nicht nur informieren, sondern den Nutzer durch eine realistische Prüfung führen:

```text
Problem erkennen
-> Realitaet einordnen
-> Risiken verstehen
-> Kosten, Einkommen oder Struktur prüfen
-> passenden nächsten Schritt waehlen
```

Das ist exakt passend für Italia Money Radar, weil das Projekt keine reine Content-Seite ist, sondern ein Orientierungs- und Prüfsystem für einen möglichen Italien-Neustart.

# 4. Was gegenüber einer einfachen Artikelvorlage verbessert wird

## 4.1 Entscheidungslogik statt nur Artikelstruktur

Nicht nur:

```text
Kurzfazit
Hauptinhalt
Checkliste
```

Sondern zusaetzlich:

```text
Realistische Einordnung
Entscheidungsmatrix
Risikoampel / Risikoabschnitt
Was du nicht daraus ableiten solltest
Nächster sinnvoller Schritt je nach Situation
```

## 4.2 Artikel werden als Prüfseiten gedacht

Jeder Artikel soll beim Nutzer dieses Ergebnis erzeugen:

```text
Ich habe nicht nur etwas gelesen.
Ich weiss jetzt besser, was ich prüfen muss.
```

## 4.3 Sensibler Themenfilter

Da Italia Money Radar sensible Themen beruehrt, muss jeder Artikel vor falschen Aussagen geschuetzt werden:

```text
Geld
Auswanderung
Partita IVA
Regime forfettario
Resto al Sud
Online-Einkommen
Napoli / Süditalien
```

Jeder Artikel muss intern klaeren:

```text
Was kann der Artikel sicher sagen?
Was darf er nicht versprechen?
Wo braucht der Nutzer professionelle Prüfung?
Welche Formulierungen sind verboten?
```

## 4.4 Bessere Nutzerfuehrung

Jeder Artikel führt nicht nur zu irgendwelchen Links, sondern zu einer nächsten sinnvollen Entscheidung:

```text
Wenn Budget unklar ist -> Kostenrechner.
Wenn Standort unklar ist -> Napoli / Süditalien lesen.
Wenn Einkommen unklar ist -> Online-Geld lesen.
Wenn Selbststaendigkeit relevant wird -> Partita IVA lesen.
Wenn Förderung geplant ist -> Fördercheck lesen und professionell prüfen lassen.
```

## 4.5 Lesbarkeit und Barrierearmut

Das Template muss leicht lesbar bleiben:

```text
klare Überschriften
kurze Absätze
sprechende Linktexte
mobile Tabellen mit horizontalem Scrollen
keine unklaren Buttons
keine Überschriftenspruenge
keine überladenen Boxen
```

# 6. Finale Template-Bausteine

## 6.1 SEO-Head

Pflichtbestandteile:

```text
title
meta description
canonical
robots index, follow
viewport
charset
stylesheet
Article JSON-LD
```

Anforderungen:

```text
Der Title muss eindeutig sein.
Die Meta Description muss seitenindividuell sein.
Die Canonical-URL muss absolut sein.
Die Seite darf nur eine H1 haben.
Interne Links müssen crawlbar und relativ sein.
```

## 6.2 Header / Navigation

Die Navigation muss zur bestehenden Website passen:

```text
Start
Kostenrechner
Napoli
Online-Geld
Partita IVA
Fördercheck
```

Im Footer zusaetzlich:

```text
Impressum
Datenschutz
```

## 6.3 Hero

Der Hero muss sofort klaeren:

```text
Thema
Nutzen
Risiko
nächster Schritt
```

Der Hero enthält:

```text
Cluster / Kategorie
H1
2-4 Sätze Einleitung
Stand
primärer CTA
sekundärer CTA
```

Verboten:

```text
Hype
Garantien
schnell-reich-Sprache
romantisierte Italien-Sprache
Förderversprechen
Steuerersparnis-Versprechen
```

## 6.4 Kurzfazit

Direkt nach dem Hero. Maximal 4 Punkte:

```text
wichtigste Antwort
entscheidender Faktor
größtes Risiko
nächster sinnvoller Schritt
```

## 6.5 Realistische Einordnung

Pflichtabschnitt. Er erklaert, warum das Thema nicht pauschal beantwortet werden kann.

Beispiel:

```text
Viele suchen nach einer einfachen Antwort. Für einen Italien-Neustart reicht aber keine einfache Ja-/Nein-Antwort, weil Ort, Miete, Einkommen, Versicherung und Rücklagen zusammen geprüft werden müssen.
```

## 6.6 Für wen ist dieser Artikel?

Konkret, nicht allgemein. Beginn:

```text
Dieser Artikel ist relevant, wenn du ...
```

Dann 4-6 konkrete Situationen.

## 6.7 Die Kernaussage

Die Hauptaussage wird als eigener Abschnitt sichtbar gemacht. Sie muss sachlich, klar und nicht werblich sein.

## 6.8 Entscheidende Faktoren

Jeder Artikel braucht eine Faktorenliste. Sie wird je nach Artikeltyp angepasst.

## 6.9 Hauptinhalt

Der Hauptinhalt besteht aus logischen H2-Abschnitten. Jede H2 beantwortet eine echte Nutzerfrage.

Beispielhafte H2-Fragen:

```text
Warum ist dieses Thema wichtig?
Welche Faktoren entscheiden?
Was wird oft unterschaetzt?
Wann kann es realistisch werden?
Wann wird es riskant?
Was sollte vor dem nächsten Schritt geprüft werden?
```

## 6.10 Realistischer wird es, wenn ...

Dieser Abschnitt zwingt zu konstruktiver, aber vorsichtiger Orientierung.

## 6.11 Riskant wird es, wenn ...

Dieser Abschnitt ist Pflicht, weil Italia Money Radar keine Illusionen verkaufen darf.

## 6.12 Häufige Denkfehler

Jeder Artikel sollte falsche Annahmen entschärfen.

Beispiele:

```text
Süditalien ist guenstig, also reicht wenig Geld.
Online-Einkommen entsteht schnell.
Partita IVA spart automatisch Steuern.
Förderung kann ich sicher einplanen.
Napoli ist automatisch die beste Loesung.
```

## 6.13 Risiken und Grenzen

Pflichtbox. Sie erklaert:

```text
was der Artikel nicht leisten kann
welche Unsicherheiten bestehen
warum individuelle Prüfung nötig sein kann
welche Entscheidungen nicht allein aus dem Artikel abgeleitet werden sollten
```

## 6.14 Checkliste

Mindestens 6 konkrete Prüffragen. Keine Allgemeinplaetze.

## 6.15 Entscheidungsmatrix

Pflicht. Spalten:

```text
Situation
Einschätzung
Nächster Schritt
```

Beispiel:

| Situation | Einschätzung | Nächster Schritt |
|---|---|---|
| Budget unklar | zu frueh | Kostenrechner nutzen |
| Einkommen unsicher | riskant | Online-Einkommen prüfen |
| Steuerstatus unklar | sensibel | professionelle Prüfung |

## 6.16 Nächster sinnvoller Schritt

Kein generisches Ende. Jeder Artikel endet mit einer klaren internen Handlung.

## 6.17 Passende nächste Seiten

Mindestens 3 interne Links:

```text
1 Link zum Kostenrechner oder Fördercheck
1 Link zur passenden Hauptseite
1 Link zum nächsten logischen Thema
```

Keine Linktexte wie:

```text
hier klicken
mehr erfahren
weiterlesen
```

Stattdessen:

```text
Italien-Kostenrechner oeffnen
Online-Einkommen in Italien realistisch prüfen
Napoli-Neustart ansehen
Partita IVA Orientierung lesen
Fördercheck oeffnen
```

## 6.18 Hinweis / Disclaimer

Pflicht. Sichtbar. Nicht versteckt.

Basis-Hinweis:

```text
Hinweis: Diese Seite bietet allgemeine Orientierung und ersetzt keine Rechtsberatung, Steuerberatung, Finanzberatung, Förderberatung oder Immobilienberatung. Programme, Kosten, Voraussetzungen und persoenliche Situationen können sich aendern.
```

Anpassung je Artikeltyp:

```text
Kostenartikel: keine Finanzberatung
Standortartikel: keine Immobilienberatung / keine Standortgarantie
Online-Einkommen: keine Einkommensgarantie
Partita IVA / regime forfettario: keine Steuer- oder Rechtsberatung
Resto al Sud: keine Förderberatung / keine Fördergarantie
```
